    EML Publication Model for OW1 Dataset Metadata

    Ecological Metadata Language (EML) structure defining how Rutgers OW1 Bottom Trawl Survey dataset-level metadata should be published. This schema defines EML elements with mappings back to source OW1 DatasetMetadata fields. Uses SKOS mapping predicates in reverse (EML elements map to OW1 fields) to document both semantic relationships and publishing structure.

    Schema file: models/datasets/rutgers/ow1-to-eml-mappings.yaml

    Purpose

    EML provides standardized dataset-level metadata for the Darwin Core Archive.

    Metadata Mappings

    ResponsibleParty

    Information about a person or organization responsible for the dataset. Used for creator, contact, and associatedParty roles.

    EML Element Source Field Notes
    individual_name Generated
    organization_name Generated
    position_name Generated
    address Generated
    phone Generated
    electronic_mail_address Generated
    online_url Generated
    role contributor_role Map from contributors list with roles

    Project

    Information about the research project that produced the dataset.

    EML Element Source Field Notes
    title dataset_title Direct mapping from dataset_title
    personnel contributors Map from contributors list
    abstract summary Direct mapping from summary field
    funding acknowledgement Direct mapping from acknowledgement field

    Coverage

    Information about the geographic, temporal, and taxonomic coverage of the dataset.

    EML Element Source Field Notes
    geographic_coverage Generated
    temporal_coverage Generated Requires temporal extent data from station/tow tables
    taxonomic_coverage species_common_name, species_scientific_name_lookup, itis_tsn Aggregate unique species from all CatchRecords

    GeographicCoverage

    Geographic area(s) covered by the dataset.

    EML Element Source Field Notes
    geographic_description Generated Describe survey area: OW1 Wind Farm lease area and control sites
    bounding_coordinates Generated Requires coordinate data from station/tow tables

    BoundingCoordinates

    Geographic bounding box defined by coordinates.

    EML Element Source Field Notes
    west_bounding_coordinate Generated
    east_bounding_coordinate Generated
    north_bounding_coordinate Generated
    south_bounding_coordinate Generated

    TaxonomicCoverage

    General taxonomic scope of the dataset, describing the major taxonomic groups covered.

    EML Element Source Field Notes
    general_taxonomic_coverage Generated Example: 'Bottom-dwelling fish and invertebrates from the Mid-Atlantic Bight'

    Methods

    Information about the methods used to collect or process the data.

    EML Element Source Field Notes
    method_step comment Document sampling methodology from comment field
    sampling Generated Describe: 20 tows in lease area, 20 in control area per season
    quality_control Generated Document QA/QC procedures if available

    Distribution

    Information about how the dataset is distributed or accessed.

    EML Element Source Field Notes
    online Generated

    Online

    Information about online access to the dataset.

    EML Element Source Field Notes
    online_url Generated
    online_description Generated Describe ERDDAP access or other distribution methods
